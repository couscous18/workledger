# Release Notes v0.1.0

`workledger` v0.1.0 is the first public release of the trace-to-work layer:

- raw traces become human-reviewable `WorkUnit`s
- evidence and uncertainty stay attached
- review queues remain visible instead of being flattened away
- comparative economics stay explicit and editable
- downstream publication bundles can extend from the same canonical work layer

PyPI is not published yet. The supported public install path for this release is source install from GitHub.

## Try This In 60 Seconds

```bash
git clone https://github.com/couscous18/workledger.git
cd workledger
uv sync --all-extras
uv run wl demo open-traces --project-dir .workledger/open-traces --open-report
uv run wl compare-costs --from-project .workledger/open-traces
```

## What Ships In v0.1.0

- canonical models for `ObservationSpan`, `WorkUnit`, `ClassificationTrace`, `PolicyDecision`, `EvidenceRef`, `PolicyPack`, and `ReportArtifact`
- rollup that compresses low-level spans into accountable units of work
- local DuckDB-backed storage plus JSON, CSV, Parquet, Markdown, and HTML outputs
- explainable policy packs for management reporting, software CapEx review, IFRS, US GAAP, and tax R&D
- a local FastAPI server and published OpenAPI artifact
- runnable demos, a proof artifact, comparative economics reporting, and a Hugging Face-ready downstream software capex bundle
