# workledger v0.1.0

`workledger` is an agent work ledger for AI systems. It turns raw traces into reviewable `WorkUnit`s with evidence, policy context, and transparent economics.

## Summary

This is the first public release of `workledger`. It packages the core local-first ledger model, ingestion pipeline, rollup engine, policy-backed classification flow, reporting outputs, and documentation into an alpha OSS release that teams can try from source today.

## Key Features In This Version

- Core models for `ObservationSpan`, `WorkUnit`, `ClassificationTrace`, `PolicyDecision`, `EvidenceRef`, `PolicyPack`, and `ReportArtifact`
- Ingestion for JSONL, OpenInference-like payloads, OTEL JSON, CloudEvents JSON, and SDK-shaped events
- Rollup that compresses low-level spans into business-level work units
- Declarative policy packs with explainable rule matches and review-required states
- DuckDB-backed local store with JSON, CSV, and Parquet export plus terminal, Markdown, JSON, CSV, Parquet, and HTML reports
- `wl` CLI, local FastAPI server, runnable demos, JSON Schema export, and published OpenAPI artifacts

## Quick Install And Try

PyPI distribution is not live for this release. Install from source from this repository:

```bash
git clone https://github.com/couscous18/workledger.git
cd workledger
uv sync --all-extras
uv run wl demo agent-cost --project-dir .workledger/agent-cost --open-report
uv run wl compare-costs --from-project .workledger/agent-cost
```

## Alpha Status

`workledger` v0.1.0 is an alpha release. It is intended to make the project publicly usable and reviewable, not to promise stable interfaces yet.

## Known Limitations

- Alpha quality: rough edges are expected
- Breaking changes may happen before a stable `1.0.0`
- Local or source install may be required until package distribution is live
